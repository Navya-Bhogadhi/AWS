Cloud Computing

Deployment Models

         The AWS Cloud offers three cloud deployment models: Cloud, Hybrid, and On-Premises. 
       1. Cloud-based applications are fully deployed in the cloud and do not have any parts that run on premises.
       2. A hybrid deployment connects infrastructure and applications between cloud-based resources and existing resources that are not in the cloud, such as on-
 premises resources.  
       3. However, a hybrid deployment is not equivalent to an on-premises deployment because it involves resources that are located in the cloud.
Cloud and Virtualization Summaries

Cloud

Scenario What Happens? Example
Run all parts of an app in the cloud The whole app is cloud-based Gmail, Netflix, Spotify
Migrate existing apps to the cloud Move old software to the cloud Banks moving databases to AWS
Design and build new apps in the cloud Create apps using cloud-native tools Uber, Zoom

On-Premises Infrastructure

Scenario Solution Example
Connecting cloud and on-premises servers Use VPN, Direct Connect, or ExpressRoute A bank connects its private servers to cloud AI services.
Integrating cloud with old IT systems Use APIs or hybrid cloud platforms A retailer connects its old inventory system to cloud-based analytics.

Hybrid

Scenario Solution Example
Deploying applications efficiently Use virtual machines (VMs) and resource management tools A company runs multiple applications on one server using VMware or Hyper-V.
Optimizing resource utilization Use containers instead of VMs to share resources A company runs Docker & Kubernetes to maximize server capacity.
